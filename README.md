# CS320-Software-Test-Automation
CS320 Portfolio

How can I ensure that my code, program, or software is functional and secure? 

I can ensure my software is functional and secure by thoroughly testing it throughout the development process instead of waiting until the end. In this course, I wrote JUnit
testes to verify that each class and method behaved as expected, including handling invalid inputs and edge cases. Writing these tests helped me find issues early and confirmed 
that my code met the project requirements. Security also starts with validating data, enforcing constraints, and designing code that prevents invalid information from being accepted.

How do I interpret user needs and incorporate them into a program? 

I begin carefully regiewing the project requirements to understand what the user expects the software to do. I break those requirements into smaller pieces and implement each feature
one at a time. Throughout this course, I used the requirements to guide both my code and my unit tests, making sure every requirement was covered. This approach helped ensure that the
final program met the user's needs while remaining reliable. 

How do I apprach designing software?

I approach software design by planning before I begin coding. I like to break a problem into smaller, manageable components and give each class a specific responsibility. As I build
the program, I continuously test my work to make sure each piece functions correctly before moving on. This course reinforced the importance of writing clean, organized, and maintainable
code while using testing as part of the overall design process rather than treating it as a final step.
